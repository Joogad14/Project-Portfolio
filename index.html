About

Task Scheduler (Queue-Based): This project simulates how tasks are handled using a FIFO queue. Tasks are added to the end of the queue (enqueue) and processed from the front (dequeue). The class also provides helper functions to view the next task (front), the last task (end), check if the queue is empty, and print the current task list.

Undo–Redo System (Stack-Based): This project implements an Undo–Redo system using two stacks. Every new action is stored in the undo stack, while undone actions are moved into the redo stack. This mimics how text editors and drawing apps manage editing history.

Unique Visitor Tracker (Set-Based): This project implements a unique visitor tracker using a Set. Every new visitor ID is stored in the set, ensuring duplicates are automatically ignored. This mimics how websites track unique visitors using identifiers like email or IP address.
